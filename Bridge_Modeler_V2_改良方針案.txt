Bridge Modeler V2 改良方針案

## 結論

この機能は、現在の6ステップを継ぎ足すより、「橋梁モデル作成 V2」として機能単位で作り直すのが妥当です。

ただし、プロジェクト全体をゼロから作り直す必要はありません。作り直す対象は主に次です。

- 橋梁モデル作成画面
- 橋梁専用データモデル
- LINERから橋梁区間を抽出する処理
- 桁・支承・横桁・節点・部材の生成パイプライン
- 荷重載荷領域とFEM骨組みの対応処理

一方、既存の以下は可能な限り再利用します。

- LINERの平面・縦断・横断座標計算
- 測点変換
- 横断勾配・幅員変化の評価
- 3D Viewer
- FEMソルバー
- DrawingDocument
- 平面図・縦断図・横断図・DXF
- プロジェクト保存基盤

つまり、全面改修ではなく「橋梁モデラー部分だけを新しいドメインで作り直す」のが最も安全です。

---

# 1. 現在の画面で問題になっている点

画像を見る限り、現在の画面は「簡易な一径間橋を自動生成するウィザード」としては成立していますが、実橋梁の設計フローとは構造が合っていません。

## Step 1「道路条件」

現在は車線数、車線幅、歩道幅、高欄幅などを入力しています。

しかし、これらはすでにLINER側で設定されている道路情報と重複します。橋梁側では、道路断面を再入力するのではなく、

- LINERのどの路線を使うか
- どの測点区間を橋梁とするか
- 道路断面のうち、どこまでを橋面として採用するか
- 壁高欄や中央分離帯を構造上どう扱うか

を設定すべきです。

したがって、現在のStep 1は廃止し、「路線・橋梁区間」画面に置き換えるべきです。

---

## Step 2「支間設定」

現在は「支間長30m」を入力していますが、実務上は橋台・橋脚の位置を測点で設定し、その結果として支間長が決まります。

現在の方式：

支間長を入力
↓
橋脚位置を推定

改良後：

A1、P1、P2、A2の測点を指定
↓
各支間長を自動計算

とすべきです。

---

## Step 3「衝撃係数」

衝撃係数は橋梁形状を決める初期ステップではありません。

少なくとも、

- 橋梁区間
- 支間割り
- 桁構成
- 活荷重方式
- 適用基準

が決まった後に扱う項目です。

したがって、独立したStep 3から外して、後段の「活荷重・解析条件」へ移動させるべきです。

---

## Step 5「荷重設定」

現在の荷重設定は、FEM部材に直接荷重を紐付ける構造に見えます。

しかし、道路橋の活荷重では次を分離する必要があります。

1. 車両が走行可能な橋面領域
2. 輪荷重が移動する走行軌跡
3. 荷重を構造部材へ分配するための床版・格子
4. 実際のFEM骨組み

したがって、このソフトでも、

道路・橋面形状
≠ 輪荷重載荷面
≠ FEM骨組み

として分離する必要があります。

---

# 2. 新しい機能の基本構成

次の4層に分離するのがよいです。

## 第1層：道路線形モデル

既存LINERが管理します。

RoadAlignment
├─ 平面線形
├─ 縦断線形
├─ 横断勾配
├─ 道路幅員
├─ 車道端
├─ 歩道
├─ 中央分離帯
└─ 測点情報

道路全体のデータであり、橋梁モデル作成機能から直接書き換えないものとします。

---

## 第2層：橋梁区間モデル

LINERから一部区間を参照します。

BridgeProject
├─ sourceAlignmentId
├─ sourceRevision
├─ startStation
├─ endStation
├─ localStationOrigin
├─ deckSections
├─ barriers
├─ medianZones
├─ trafficLoadZones
└─ supportLines

重要なのは、線形をコピーして独立させるのではなく、原則として

LINERの路線ID
＋
開始測点
＋
終了測点
＋
参照時のrevision

を保存することです。

LINERを変更した場合には、

- 更新を取り込む
- 現在の橋梁モデルを維持する
- 差分を確認する

を選択できるようにします。

---

## 第3層：橋梁構造モデル

BridgeStructure
├─ abutments
├─ piers
├─ bearingLines
├─ bearings
├─ girderLines
├─ crossGirders
├─ diaphragms
├─ deckModel
├─ materials
└─ sectionProperties

橋台・橋脚・桁・横桁をここで定義します。

---

## 第4層：解析モデル

AnalysisModel
├─ nodes
├─ members
├─ springs
├─ constraints
├─ loadCases
├─ movingLoadSurface
├─ combinations
└─ solverSettings

構造モデルから自動生成されるものです。

ユーザーが節点番号や部材番号を手作業で入力するのではなく、橋梁構造モデルから生成します。

---

# 3. 推奨する新しい画面構成

現在の6ステップは、次の9ステップ程度に再編するのがよいです。

## Step 1：路線・橋梁区間

設定項目：

- LINERプロジェクト
- 基準ライン
- 橋梁開始測点
- 橋梁終了測点
- 橋梁内ローカル原点
- 線形revision
- 橋梁区間の方向

入力例：

基準ライン：L1
開始測点：No.25+35.000
終了測点：No.31+15.000
橋長：580.000m

平面図と縦断図で範囲を色付けし、図上をクリックして開始・終了位置を設定できるようにします。

---

## Step 2：橋面・横断構成

ここでは道路断面を再入力せず、LINERの断面を橋梁用に分類します。

例：

左壁高欄
左歩道
左車道
中央分離帯
右車道
右歩道
右壁高欄

各横断要素にstable IDを持たせます。

deck-left-edge
barrier-left
lane-left-1
median
lane-right-1
barrier-right
deck-right-edge

横断オフセットの並びは、signed offset順ではなく、ユーザーが定義した編集順序を正とします。

---

## Step 3：輪荷重載荷領域

ここが現在の機能に不足しています。

輪荷重が載る範囲は、単なる車線数ではなく、横断方向と橋軸方向の領域として管理します。

TrafficLoadZone
├─ startStation
├─ endStation
├─ leftOffset
├─ rightOffset
├─ excludedZones
├─ laneDefinitions
└─ vehicleDirection

画面では次を同時表示します。

- 平面図：橋軸方向の載荷区間
- 横断図：横断方向の載荷可能範囲
- 中央分離帯・壁高欄・歩道：載荷除外領域
- 車線中心
- 想定輪軌跡

最初のMVPでは、

左載荷限界
右載荷限界
中央分離帯除外
歩道除外

まで実装し、その後にT荷重・L荷重・車線載荷へ発展させるのがよいです。

---

## Step 4：橋台・橋脚・支承線

表形式と平面図操作を併用します。

名称  種別    測点          斜角     備考
A1    橋台    No.25+35      90°      起点側
P1    橋脚    No.27+10      82°
P2    橋脚    No.29+00      78°
A2    橋台    No.31+15      90°      終点側

支間長は自動計算します。

A1-P1 = 175m
P1-P2 = 190m
P2-A2 = 215m

支承線は、

- 基準ライン直角
- 任意斜角
- 既存横断線参照
- 2点指定

に対応できる設計にします。

---

## Step 5：主桁・横桁配置

主桁は「本数と間隔」だけでなく、ラインとして定義します。

GirderLine
├─ id
├─ label
├─ referenceAlignment
├─ startStation
├─ endStation
├─ offsetDefinition
├─ verticalOffset
├─ sectionAssignment
└─ continuity

入力方式は複数用意します。

- 等間隔配置
- オフセット一覧
- 既存道路ラインを参照
- 始端・終端で異なるオフセット
- 拡幅に追従
- 任意の折れ線

横桁は、

- 支承線上
- 一定間隔
- 測点指定
- 主桁断面変化位置
- 任意追加

から自動生成します。

---

## Step 6：部材・断面・支承条件

設定内容：

- 主桁断面
- 横桁断面
- 床版剛性
- 材料
- 支承形式
- ばね値
- 固定・可動方向
- 部材端結合
- 剛域
- オフセット

ここで橋梁断面テンプレートを使用します。

例：

- 鈑桁橋
- 箱桁橋
- PC箱桁
- 合成桁
- 一本棒全体系モデル
- 平面格子モデル
- 立体格子モデル

---

## Step 7：FEMメッシュ生成

現在の「FEMモデル生成」を単一ボタンにせず、生成規則を確認できる画面にします。

節点を作成する位置は、次の和集合とします。

橋梁開始・終了
橋台・橋脚・支承線
横桁位置
断面変化位置
横断構成変化位置
線形要素変化位置
荷重領域境界
ユーザー指定分割点
最大部材長による自動分割点

各測点で、

1. 中心線XYZを計算
2. 横断方向ベクトルを計算
3. 横断勾配を評価
4. 各主桁オフセット位置を計算
5. 節点を生成
6. 主桁部材を接続
7. 横桁・支承・ばねを生成

します。

生成後には必ず診断を表示します。

節点数
部材数
未接続節点
重複節点
ゼロ長部材
支承未設定
断面未設定
荷重領域外の部材
桁と支承線の未交差

---

## Step 8：荷重・解析条件

現在の衝撃係数はここへ移します。

設定内容：

- 自重
- 舗装
- 壁高欄
- 中央分離帯
- 添架物
- 温度荷重
- 支点沈下
- 風荷重
- 活荷重
- 衝撃係数
- 荷重組合せ
- 固有値解析
- 応答スペクトル解析

自重は部材属性から生成し、壁高欄などは横断要素から線荷重へ変換します。

---

## Step 9：解析・結果・図面

同一画面に全部詰め込まず、タブを分けます。

解析実行
構造図
変形図
断面力図
反力
固有モード
平面一般図
縦断一般図
橋梁断面構成図
格子図
荷重載荷図
DXF出力
帳票

断面構成図には最低限、

- 橋面幅
- 壁高欄
- 中央分離帯
- 車道・歩道
- 輪荷重載荷範囲
- 主桁位置
- 横桁
- 床版厚
- 部材名称
- 寸法線

を表示します。

# 4. UIはウィザードより「橋梁モデラー」にする

現在のような完全な順送りウィザードでは、後から橋脚を移動したときに桁・メッシュ・荷重を確認しにくくなります。

推奨レイアウトは次です。

┌──────────────────────────────────────────┐
│ 橋梁名 / 基準路線 / 橋梁区間 / 保存 / 解析    │
├──────────┬────────────────┬───────────────┤
│ 設定ツリー │ 入力・編集パネル    │ プレビュー      │
│          │                    │ 平面            │
│ 1 区間   │                    │ 縦断            │
│ 2 横断   │                    │ 横断            │
│ 3 荷重面 │                    │ 3D              │
│ 4 下部工 │                    │ FEM             │
│ 5 桁     │                    │                 │
└──────────┴────────────────┴───────────────┘

現在の左右2ペイン構成は残せますが、右側に常時プレビューを置く方がよいです。

特に、

- 橋台・橋脚位置
- 桁位置
- 輪荷重領域

は、表入力だけでなく図上編集できるようにするべきです。

# 5. 作り直す部分と残す部分

## 作り直すべき部分

- 現在の6ステップのデータ型
- lane countから直接桁位置を算出する処理
- 支間長から橋脚位置を決める処理
- 単純な荷重→line_id割当
- 一括FEM生成処理
- bridge_idを各画面に繰り返し表示する構成
- 道路断面と橋梁断面を同一視する設計

## 残す候補

- 画面全体のテーマ
- ダイアログ／モーダル構成
- LINER計算エンジン
- 測点表記
- 横断オフセットstable ID
- 3D Viewer
- FEMソルバー
- 既存FEMモデル形式
- DrawingDocument
- DXF exporter
- プロジェクト保存API

# 6. 推奨する開発方法

既存画面を直接壊すのではなく、新しい経路を作るのが安全です。

/bridge-modeler          旧機能
/bridge-modeler-v2       新機能

開発中は旧機能を残します。

新V2では、最初に次だけ実装します。

## MVP 1

- LINER路線選択
- 橋梁開始・終了測点
- A1・P1・A2配置
- 主桁オフセット配置
- 横桁自動生成
- FEM節点・部材生成
- 3D表示
- 保存・再読込

## MVP 2

- 壁高欄・中央分離帯
- 輪荷重載荷領域
- 自重・線荷重
- 支承条件
- FEM実行
- 断面力表示

## MVP 3

- 移動荷重
- 影響線
- 荷重組合せ
- 断面構成図
- 正式図面・DXF
- 帳票

# 7. 最終的な推奨判断

現在の画面を継ぎ足して完成させるのは勧めません。

理由は、見た目の問題ではなく、現在のデータ構造が、

道路条件
→ 支間長
→ 衝撃係数
→ ライン
→ 荷重
→ FEM

という簡易モデル向けの流れになっているためです。

本来必要なのは、

LINER道路線形
→ 橋梁区間抽出
→ 橋面・載荷領域
→ 橋台・橋脚・支承線
→ 主桁・横桁
→ 断面・支承条件
→ FEMメッシュ
→ 荷重・解析
→ 結果・図面

です。

したがって方針としては、

「既存機能をLegacyとして残し、Bridge Modeler V2を新しいデータモデルから構築する。ただしLINER、Viewer、FEMソルバー、Drawing/DXFは再利用する。」

これが最も事故が少なく、将来の複雑な橋梁にも耐えられる構成です。
