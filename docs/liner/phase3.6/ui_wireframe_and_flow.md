# Phase 3.6 UI Wireframe and Flow

## 0. UI 方針

Phase 3.6 は PDF 写経入力に最適化した独立ウィザードである。Phase 3.5 の 6 タブとは同じ階層に置かず、メニューから別画面群として起動する。

基本導線:

```text
プロジェクト -> 橋梁 -> 基準ライン/橋軸線マスタ -> 横断面リスト -> 横断面編集 -> ピア/スパン確認 -> Phase 3.5 draft エクスポート
```

## 1. トップ: プロジェクト一覧

```text
+--------------------------------------------------------------+
| JIP-LINER PDF 入力                                           |
+--------------------------------------------------------------+
| [新規プロジェクト] [名前を付けて保存] [保存済みから開く]      |
| [既存JSONを開く] [最近使ったファイル]                         |
|                                                              |
| プロジェクト名        橋梁数  最終更新      状態              |
| Hランプ4号橋          1       2026-07-02    入力中            |
| ...                                                          |
+--------------------------------------------------------------+
```

PDF で見る場所:

- プロジェクト名: PDF 表紙、設計条件資料、帳票ヘッダ
- 橋梁名: 各横断面ページのヘッダ
- Hランプ4号橋サンプル: `REPORT09_2編-01_Hランプ4号橋_線形計算書.PDF` の 12 ページ目以降

## 2. 橋梁基本情報

```text
+--------------------------------------------------------------+
| 橋梁基本情報                                      [わからん] |
+--------------------------------------------------------------+
| 橋梁名           [ Hランプ4号橋                         ]    |
| 路線/ランプ名    [                                      ]    |
| 橋梁形式         [単純橋 v] [連続橋] [上下線一体] [分離]     |
| 水平座標系       測地系 [JGD2011] epoch [任意] 系番号 [ ]    |
| 鉛直座標系       標高基準 [T.P.] ジオイド [任意]             |
|                                                              |
| [戻る]                                      [保存して次へ]   |
+--------------------------------------------------------------+
```

PDF で見る場所:

- 橋梁名、方位角、累加距離: 横断面ページのヘッダ
- 座標系: 設計条件資料または成果品共通条件

`わからん` ヘルプ:

- 座標系が不明でも入力を継続できるが、エクスポート時に warning を出す。
- epoch とジオイドは不明なら空欄可。必須化は M7 の確認事項。

PDF で見る場所:

コピペ支援:

- Excel / PDF から行ラベルを貼り付け、1 行 1 橋軸線として展開する。

## 4. 横断面リスト

```text
+--------------------------------------------------------------+
| 横断面リスト                                      [わからん] |
+--------------------------------------------------------------+
| [横断面追加] [前ページ複製] [PDFページ番号で一括作成]        |
|                                                              |
| Page  測点      方位角       入力率  診断     操作           |
| 1     12+34.567  PDFから転記  80%     Warning  [編集][複製]  |
| 2     12+44.567  PDFから転記  10%     -        [編集][複製]  |
|                                                              |
| [戻る]                                      [保存して次へ]   |
+--------------------------------------------------------------+
```

PDF で見る場所:

- 1 ページ = 1 横断面
- ヘッダの方位角、累加距離、測点
- 末尾の交角、測点
- Hランプ4号橋サンプルでは、§3.1 大座標の横断面ページを section として登録する

前ページコピー:

- 橋軸線行、sourceRef の page 以外の列定義、未入力セル構造を複製する。
- 数値セルは初期設定で空欄。ユーザーが明示した場合だけ前ページ値を複製する。

`PDFページ番号で一括作成` ボタンの動作:

- 開始ページと終了ページを入力すると、その範囲で空の Section を生成する。
- 初期値は全セル空、azimuth と stationingRef は null、sourceRef の pdfPage のみ設定する。
- 橋軸線（points）は現在の girderLineSet のマスタから空 Point を生成する。
- Flags と診断は生成後に入力率 0% として表示する。

## 5. 横断面編集

```text
+--------------------------------------------------------------------------------+
| 横断面編集 PDF p.12                                           [わからん]       |
+------------------------------------------+-------------------------------------+
| ヘッダ                                   | 右ペイン                            |
| 方位角 [  deg ][ min ][ sec ]            | +-------------------------------+   |
| 累加距離 [            ] m                | | 横断図 SVG プレビュー          |   |
| 測点     [            ]                  | | 入力点、未定義、警告を表示      |   |
|                                          | +-------------------------------+   |
| 本体表                                   | +-------------------------------+   |
| 行 橋軸 X      Y      高さ 勾配 単距 ... | | 平面プレビュー                 |   |
| CL [   ][   ][   ][   ][   ][   ] ...    | | 前後 section の位置関係        |   |
| G1 [   ][   ][   ][   ][   ][   ] ...    | +-------------------------------+   |
| G2 [********] ...                        |                                     |
|                                          |                                     |
| [********入力] [前ページからコピー]       |                                     |
| [PDF表を貼り付け] [即時診断]             |                                     |
| [戻る] [保存] [次ページ]                 |                                     |
+--------------------------------------------------------------------------------+
```

PDF で見る場所:

- ヘッダ: 橋梁名、方位角、累加距離
- 本体: 15 行 × 9 列程度の座標、計画高、勾配、幅員データ
- 末尾: 交角、測点
- Hランプ4号橋サンプルでは、`横断面 1 : PH12(PE10)` のようなページをそのまま 1 画面として入力する

`********入力`:

- 選択セルを `null` + `flags.notComputed = true` にする。

`PDF表を貼り付け`:

- タブ区切りまたは空白区切りのテキストを既存行列へ流し込む。
- 解析できないセルは未確定状態でハイライトする。

## 7.5 名前付き途中保存 / 再開フロー

保存ダイアログ:

```text
+--------------------------------------------------------------+
| 名前を付けて保存                                             |
+--------------------------------------------------------------+
| 保存名     [ 横断面12まで入力                         ]      |
| 保存先     [ C:\...\importer-projects\             ][参照]   |
| メモ       [                                             ]    |
|                                                              |
| [ ] 既存ファイルを上書きする                                 |
| [キャンセル]                         [名前を付けて保存]      |
+--------------------------------------------------------------+
```

開くダイアログ:

```text
+--------------------------------------------------------------+
| 保存済みから開く                                             |
+--------------------------------------------------------------+
| 保存名              savedAt              lastEditedStep 操作 |
| 横断面12まで入力    2026-07-02 20:30     sectionEdit   [開く]|
| 橋梁基本情報のみ    2026-07-02 19:10     bridge        [開く]|
|                                                              |
| [既存JSONを参照]                                      [閉じる]|
+--------------------------------------------------------------+
```

再開時の動作:

- JSON 読み込み後、`savedSnapshots[].lastEditedStep` に対応する画面へジャンプする。
- `lastEditedRef.bridgeId` / `lastEditedRef.sectionId` がある場合は対象橋梁・横断面を開く。
- 未確定入力、validation error、`********` flags は保存時のまま復元する。
- Draft 保存の場合はプロジェクト一覧と画面ヘッダに `isDraftSave` バッジを表示する。

自動バックアップとの関係:

- 名前付き保存はユーザー操作で行う完全な JSON 保存である。
- 自動バックアップは Silent に実行し、名前付き保存の一覧とは別系統で管理する。
- 自動バックアップ間隔、保持数、復元 UI は実装フェーズで決める。

## 8. 状態遷移

```text
DraftCreated
  -> EditingBridge
  -> EditingLineMaster
  -> EditingSectionList
  -> EditingSection
  -> ReviewingSpanPier
  -> ReadyToExport
  -> Exported

Any Editing State -> AutoBackupSaved
Any Editing State -> ValidationBlocked
ValidationBlocked -> EditingSection
ReadyToExport -> ExportFailed
ExportFailed -> ReadyToExport
```

保存タイミング:

- 入力中: section 単位で即時保存。
- 画面遷移時: current step を保存。
- 自動バックアップ: 実装フェーズで間隔を決める。
- エクスポート時: importer JSON と conversion log を保存。

## 9. UI 分離方法

- Phase 3.5 の `liner.setup` タブ群には入れない。
- メニューまたは独立 route から起動する。
- 画面タイトル、保存 JSON、diagnostics、変換ログを Phase 3.6 専用にする。
- Phase 3.5 の確認図は変換後 pipeline の確認に使う。Phase 3.6 の右ペインプレビューは写経確認用であり、同じ画面ではない。
