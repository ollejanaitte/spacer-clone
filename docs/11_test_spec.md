# 11 Test Specification

## 1. 目的

MVPの解析精度、API契約、UI表示、エラー処理を検証するための最小限のテスト仕様を定義する。後続実装では、この文書の必須ケースを自動テスト化する。

## 2. 共通基準

- 単位はSI固定。
- 理論値比較の標準相対誤差は `1e-5`。
- 0近傍では絶対誤差を明示する。
- 符号規約はテスト内コメントに明記する。
- `NaN`、`Infinity` は失敗とする。
- 解析失敗ケースでは成功結果を返してはならない。

## 3. 必須解析検証ケース

### Case 1: 片持梁の先端集中荷重

モデル:

- 1部材。
- I端固定、J端自由。
- J端に鉛直集中荷重 `P`。

期待:

- 先端変位 `delta = P L^3 / (3 E I)`。
- 先端回転 `theta = P L^2 / (2 E I)`。
- 固定端反力 `P`。
- 固定端モーメント `P L`。

### Case 2: 単純梁の中央集中荷重

モデル:

- 中央節点を持つ単純梁。
- 両端鉛直支持。
- 剛体変位を防ぐため、必要な軸方向拘束を追加する。
- 中央節点に集中荷重 `P`。

期待:

- 中央変位 `delta = P L^3 / (48 E I)`。
- 反力 `P / 2`。
- 最大曲げ `P L / 4`。

### Case 3: 単純梁の等分布荷重

モデル:

- 中央節点を持つ単純梁。
- 全長に等分布荷重 `w`。

期待:

- 中央変位 `delta = 5 w L^4 / (384 E I)`。
- 反力 `w L / 2`。
- 最大曲げ `w L^2 / 8`。

### Case 4: 3D片持梁のねじり

モデル:

- X方向の片持梁。
- 自由端に局所X軸まわりのねじりモーメント `T`。

期待:

- ねじり角 `phi = T L / (G J)`。
- `G = E / (2(1+ν))` で計算される。
- 固定端ねじり反力 `T`。

### Case 5: 支点不足エラー

モデル:

- 節点、部材、材料、断面は妥当。
- 支点なし、または剛体運動を拘束できない。

期待:

- `MODEL_UNSTABLE` または `SOLVER_ERROR`。
- 成功結果として扱わない。
- `analysisSummary.status` は `failed`。

### Case 6: 不正な部材参照エラー

モデル:

- 部材が存在しない節点IDを参照する。

期待:

- 解析前検証で `INVALID_REFERENCE`。
- `path` が不正フィールドを指す。
- solverを実行しない。

### Case 7: 剛体モード検出

モデル:

- 全節点自由。
- 支点なし。

期待:

- `MODEL_UNSTABLE`。
- 必要に応じて `SOLVER_ERROR` を併記してよい。
- 成功結果を返してはならない。
- 変位、反力、部材端力を成功データとして返してはならない。

## 4. APIテスト

- `GET /health` が成功する。
- `POST /api/projects/validate` が正常モデルを `valid: true` にする。
- `POST /api/projects/validate` が不正参照を検出する。
- `POST /api/analysis/run` が片持梁サンプルを解析できる。
- `POST /api/analysis/run` が支点不足モデルで失敗レスポンスを返す。
- `POST /api/projects/save` がパストラバーサルを拒否する。
- `POST /api/projects/load` が存在しないファイルを適切に扱う。
- `GET /api/examples` が必須サンプルを返す。

## 5. UI/3Dテスト

- UIビルドが成功する。
- 節点表、部材表、材料表、断面表、支点表、荷重表が表示される。
- 検証エラーが画面に表示される。
- 解析成功結果の変位、反力、部材端力表が表示される。
- Three.jsビューが空モデルでクラッシュしない。
- 節点、部材、支点、荷重、変形図を表示できる。
- 選択ハイライトがUI状態と同期する。

## 6. 結果・帳票テスト

- 結果JSONが `docs/06_result_schema.md` に適合する。
- 変位CSV、反力CSV、部材端力CSVのヘッダーが仕様通り。
- 部材端力はI端、J端の2行に展開できる。
- エラー結果でも `errors` を含むJSONを返せる。
- CSV/JSONに非有限値を出力しない。

## 7. 完了条件

- 7つの必須解析検証ケースが自動テスト化されている。
- APIテストがMVP必須エンドポイントを網羅する。
- UIビルドと基本表示テストが通る。
- `docs/12_quality_gate.md` の受入基準を満たす。
