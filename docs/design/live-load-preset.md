# 活荷重プリセット設計

## 1. 目的

道路橋示方書R7を基準とした活荷重定義を、ソースコードにハードコードせず、出典付きの `LiveLoadPreset` として管理する。MVPではプリセットを解析対象にしないが、将来拡張時にT荷重、L荷重、群集荷重、A活、B活を安全に追加できる構造を定義する。

## 2. 原則

- 荷重値、係数、適用条件をPython、TypeScript、JSON Schemaへ固定しない。
- 荷重定義はプリセットデータとして読み込む。
- プリセットは版、出典、単位、適用条件を必ず持つ。
- R7以外の版も同時に保持できる。
- 解析結果には使用したプリセットIDと版を記録する。

## 3. 対象プリセット

### T荷重

床版、床組などに用いられる集中荷重系の活荷重モデルとして扱う。

設計項目:

- 軸または輪荷重の点荷重列。
- 横方向配置。
- 載荷可能範囲。
- 適用部材種別。
- A活、B活との関係。

MVPでは未実装。

### L荷重

主桁などに用いられる分布荷重系の活荷重モデルとして扱う。

設計項目:

- 等分布荷重の領域。
- 支間長や載荷長に応じるパラメータ参照。
- 車線または載荷幅との関係。
- A活、B活との関係。

MVPでは未実装。

### 群集荷重

歩道や歩行者領域に対する面荷重モデルとして扱う。

設計項目:

- 面荷重値。
- 適用領域。
- 橋種または歩道条件。
- 同時載荷ルール。

MVPでは未実装。

### A活

大型自動車の交通状況に応じた活荷重区分として扱う。荷重値そのものではなく、T荷重やL荷重の適用条件セットとして表現する。

### B活

A活と同様に、道路種別や交通条件に応じた活荷重区分として扱う。T荷重、L荷重、必要な割増や適用ルールをプリセット参照で構成する。

## 4. 版管理

`LiveLoadPreset` は以下を持つ。

- `presetId`: 永続ID。
- `edition`: 例として `R7` など。表示用文字列ではなく検索可能なキー。
- `revision`: プリセットデータの社内改訂番号。
- `effectiveDate`: 適用開始日。
- `status`: `draft`, `approved`, `deprecated`。
- `supersedes`: 置換元プリセットID。
- `checksum`: プリセット内容の監査用ハッシュ。

解析時は `presetId` と `revision` を結果へ保存し、後でプリセットが更新されても過去結果を追跡できるようにする。

## 5. 出典管理

道路橋示方書の参照位置は必ず次の形式で保持する。

```json
{
  "source": {
    "standard": "道路橋示方書・同解説",
    "edition": "R7",
    "volume": "I 共通編",
    "chapter": "",
    "section": "",
    "table": "",
    "figure": "",
    "note": ""
  }
}
```

`chapter`、`section`、`table`、`figure` は、プリセット登録時に原典確認済みの値を入れる。未確認の値を推測で埋めてはならない。

## 6. JSON構造

```json
{
  "presetId": "jra-r7-live-load-a",
  "name": "A活荷重",
  "category": "roadBridgeLiveLoad",
  "edition": "R7",
  "revision": "1.0.0",
  "status": "draft",
  "source": {
    "standard": "道路橋示方書・同解説",
    "edition": "R7",
    "volume": "",
    "chapter": "",
    "section": "",
    "table": "",
    "figure": "",
    "note": ""
  },
  "unitSystem": "kN-m",
  "parameters": [],
  "components": [
    {
      "id": "component-1",
      "type": "pointLoadTrain",
      "values": [
        {
          "name": "P",
          "value": null,
          "unit": "kN",
          "sourceRef": "source"
        }
      ],
      "layout": {}
    }
  ],
  "applicability": {
    "bridgeType": ["road"],
    "memberUse": [],
    "conditions": []
  }
}
```

`value: null` は、設計枠だけを先に持つ場合の表現である。承認済みプリセットでは必須値を未設定にしてはならない。

## 7. パラメータ式

支間長、載荷長、車線幅などに依存する荷重は、式そのものをソースコードへ埋め込まず、プリセット内の式定義として保持する。

式定義の方針:

- 式は安全な式評価器で解釈する。
- 使える変数はプリセットに明示する。
- 単位を持つ。
- 範囲条件を持つ。
- 参照出典を持つ。

```json
{
  "name": "P",
  "type": "piecewiseFormula",
  "unit": "kN/m",
  "variables": [
    { "name": "spanLength", "unit": "m" }
  ],
  "cases": [
    {
      "condition": "spanLength <= L1",
      "expression": "P1",
      "sourceRef": "source"
    }
  ]
}
```

上記の `L1` や `P1` は例示名であり、実値はプリセットデータとして登録する。

## 8. 将来拡張

- 国、機関、版の切り替え。
- ユーザー独自プリセット。
- プリセット差分表示。
- 原典確認済みフラグ。
- 承認ワークフロー。
- 設計例との照合結果。
- 解析結果へのプリセットスナップショット保存。

## 9. 禁止事項

- R7道路橋示方書の荷重値をPythonやTypeScriptへ直書きする。
- UI表示用文字列から荷重ロジックを判定する。
- プリセットIDだけを保存し、改訂番号を保存しない。
- 出典未確認の数値を承認済みとして扱う。
