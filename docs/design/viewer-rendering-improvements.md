# Viewer 描画基盤改善 設計メモ

## 1. 現状の描画課題

Viewer Phase1/Phase2 では、節点・部材・支点・荷重・変形形状・反力ラベル・部材端断面力ラベルを同一 Three.js シーン上に重ねて表示している。機能は増えたが、表示対象が多いモデルでは次の問題が出やすい。

- 部材線、荷重矢印、結果図、ラベルが重なり、読み取りにくい。
- ラベル数が増えると画面密度が高くなり、節点番号・部材番号・結果値が衝突する。
- WebGL の標準 `LineBasicMaterial.linewidth` は実行環境によって線幅が効かないため、表示サイズ UI の「部材線幅」が期待どおり見えない場合がある。
- 結果ラベルは常時同じ優先度で描画されるため、重要な数値と補助的な番号が同時に混雑する。

## 2. 部材線幅が WebGL 環境で効きにくい問題

Three.js の通常線分は、多くのブラウザ・GPU ドライバ環境で線幅 1px 相当に固定される。現在の `createLine` / `LineBasicMaterial` ベースでは、`memberLineWidth` を大きくしても見た目が変わらない環境がある。

この制約はアプリ側のバグではなく WebGL 実装依存の制限であるため、線幅を確実に制御するには別方式の線描画が必要になる。

## 3. Three.js Line2 / LineMaterial 導入案

候補は `three/examples/jsm/lines/Line2`、`LineGeometry`、`LineMaterial` の導入。

想定方針:

- 部材線・断面力図の折れ線・補助線を Line2 系へ段階移行する。
- `LineMaterial.resolution` を viewport resize 時に更新する。
- 通常線分と Line2 を共存可能にし、まず部材線だけを移行する。
- ラベル・矢印・メッシュは現行方式を維持する。

注意点:

- examples モジュール導入により bundle と型管理が少し増える。
- 線の交差、透明度、depthTest の見え方が現行と変わる可能性がある。
- テストでは JSDOM 上の WebGL 再現が難しいため、ユニットテストは geometry 生成と props 連携中心にする。

## 4. ラベル衝突回避案

### 4.1 優先度ベース非表示

ラベルに優先度を持たせ、表示密度が高い場合は低優先度ラベルから隠す。

優先度例:

1. 選択中節点・選択中部材の結果ラベル
2. 反力・断面力など解析値ラベル
3. 節点番号・部材番号
4. 補助説明ラベル

利点は実装が比較的軽いこと。欠点は「どのラベルが消えたか」がユーザーに伝わりにくいこと。

### 4.2 引き出し線案

ラベルを節点・部材端から少し離して配置し、対象点とラベルを細線で結ぶ。

利点:

- 結果値が対象位置に対応していることを保ちやすい。
- 反力ラベルや部材端断面力ラベルと相性がよい。

欠点:

- 引き出し線自体が増え、密なモデルではさらに混雑する。
- 3D カメラ回転時の配置更新が必要。

### 4.3 クラスタリング案

画面空間で近接するラベルをまとめ、代表ラベルや「+N」表示に集約する。

利点:

- 大規模モデルの初期表示が読みやすくなる。
- ズームイン時に詳細表示へ展開できる。

欠点:

- 3D 座標から画面座標への投影、カメラ変更時の再計算が必要。
- クリック・選択との整合がやや複雑。

## 5. 採用候補

短期は「優先度ベース非表示 + 選択中対象の強制表示」を採用候補とする。Line2 / LineMaterial は線幅問題の根本対策として別フェーズで導入する。引き出し線とクラスタリングは、結果ラベル数が増えてから効果を確認して採用判断する。

## 6. 実装フェーズ分割

### Phase A: 表示優先度の内部モデル化

- ラベル生成時に `userData.priority`、`userData.ownerType`、`userData.ownerId` を付与する。
- 選択中対象のラベルを優先表示する。
- UI 変更は最小限に留める。

### Phase B: Line2 / LineMaterial 試験導入

- 部材線のみ Line2 化する。
- `memberLineWidth` が見た目に反映されることを確認する。
- Electron / Chrome / Edge で手動確認する。

### Phase C: ラベル密度制御

- カメラ距離または画面上の投影密度に応じて低優先度ラベルを非表示にする。
- 選択中対象は常に表示する。

### Phase D: 引き出し線・クラスタリング検証

- 反力・部材端断面力ラベルを対象に小さく試験実装する。
- 効果が薄い場合は採用しない。

## 7. テスト方針

- ラベル優先度付与のユニットテスト。
- 表示設定から renderer へ渡る props のコンポーネントテスト。
- Line2 導入時は geometry 生成、resolution 更新、dispose 処理のテスト。
- Electron 実機で、線幅・ズーム・カメラ回転・SPACER座標系表示ON/OFFを手動確認する。

## 8. 今回実装しないこと

- Line2 / LineMaterial の導入。
- ラベル衝突回避ロジック。
- 優先度ベース非表示の実装。
- 引き出し線の実装。
- クラスタリングの実装。
- 断面力分布図、カラーマップ、モーメント円弧矢印。
