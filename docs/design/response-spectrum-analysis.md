# response-spectrum-analysis.md

## 1. 応答スペクトル解析の位置付け

応答スペクトル解析は、固有値解析結果を用いて地震時応答を推定する上位解析である。

初期実装では、基準スペクトルの自動生成は行わず、ユーザー入力のスペクトルデータを用いる。

道路橋示方書・鉄道標準等への対応は将来拡張とする。

## 2. 固有値解析との接続

応答スペクトル解析は、以下を固有値解析から受け取る。

* 固有周期
* 固有振動数
* モード形
* モード質量
* 刺激係数
* 有効質量

応答スペクトル解析単独では実行せず、内部で固有値解析を実行するか、既存の固有値解析結果を参照する。

## 3. 入力スペクトルデータ構造

入力スペクトルは、周期と応答値のペアで定義する。

MVPでは加速度応答スペクトルのみを扱う。

入力スペクトルは疑似加速度スペクトル `Sa` として扱う。

スペクトル単位は `m/s^2` 相当を基本とする。

`gal`, `g` 入力は将来対応とする。

```json
{
  "spectrumCases": [
    {
      "id": "spec-1",
      "name": "入力スペクトル",
      "valueType": "acceleration",
      "damping": 0.05,
      "points": [
        { "period": 0.0, "value": 0.0 },
        { "period": 0.1, "value": 1.0 },
        { "period": 1.0, "value": 0.5 }
      ]
    }
  ]
}
```

応答値の種類。

```text
acceleration
velocity
displacement
```

MVPでは加速度応答スペクトルを優先する。

`velocity`, `displacement` スペクトルは将来対応とする。

周期方向の補間方法は線形補間とする。

周期範囲外の扱いは未決事項とする。

候補。

```text
端点値を使う
エラーにする
```

## 4. 減衰定数

初期実装では、全モード一律の減衰定数を入力する。

```text
h = 0.05
```

将来は以下を検討する。

* モード別減衰
* 部材種別減衰
* レイリー減衰
* 構造種別による標準値テンプレート

## 5. 起震方向

起震方向は全体座標系で指定する。

初期候補。

```text
X
Y
Z
任意方向ベクトル
```

MVPでは X/Y/Z の3方向を優先する。

起震方向ベクトル `r` に対して、刺激係数と有効質量を算出する。

任意方向ベクトルは将来対応とする。

## 6. モード合成

MVPでは SRSS を優先する。

```text
R = sqrt(sum(Ri^2))
```

CQC は将来対応または Phase 2 とする。

CQC では、近接モードと減衰定数の扱いが必要となる。

SRSS 合成結果は符号付き結果ではなく、符号なし最大値・包絡値として扱う。

CQC は将来対応とする。

## 7. 出力データ構造

出力は以下を想定する。

```json
{
  "analysisType": "response_spectrum",
  "spectrumCaseId": "spec-1",
  "direction": "X",
  "combination": "SRSS",
  "modalResponses": [],
  "combinedResponse": {
    "displacements": [],
    "reactions": [],
    "memberEndForces": []
  }
}
```

出力対象。

* 各次応答変位
* 合成最大変位
* モード別寄与率
* 有効質量比

MVPでは応答変位を優先する。

反力・部材端力の応答スペクトル出力は将来対応とする。

## 9. 将来の道路橋示方書・鉄道標準対応方針

初期実装では基準値を推測しない。

将来対応では、以下を別レイヤーとして追加する。

```text
standard-spectrum-provider
```

候補。

* 道路橋示方書スペクトル生成
* 鉄道構造物等設計標準スペクトル生成
* 地盤種別
* 地域別補正
* 設計水平震度
* 減衰補正
* 固有周期範囲ごとの式

基準式・係数は、必ず正式な基準文書に基づいて実装する。

基準スペクトル生成は将来対応とする。
