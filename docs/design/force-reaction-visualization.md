# 断面力・反力可視化設計

## 1. 現状仕様

静的解析結果は `reactions` と `memberEndForces`、応答スペクトル結果は選択結果内の反力と部材断面力を返す。Viewerは反力矢印とN/My/Mzの簡易線図を表示できる。

## 2. 課題

- 数値を3D位置で確認できない。
- 部材端力の符号規約がViewer上で説明されていない。
- モーメント成分の視覚表現とローカル軸表示が未整備である。

## 3. 目標仕様

フェーズ1ではRFX/RFY/RFZと部材端FXを数値ラベルで表示する。フェーズ2以降でFY/FZ、モーメント、分布図、カラーマップを追加する。

## 4. UI案

```text
[反力表示]
  [x] 数値ラベル  [x] RFX [x] RFY [x] RFZ

[断面力表示]
  [x] 部材端ラベル [x] FX [ ] FY [ ] FZ [ ] MX [ ] MY [ ] MZ

凡例: 反力は全体座標系。部材端FXは部材ローカルx軸正方向を正とする。
```

## 5. データ構造案

API追加は行わない。静的解析は既存 `AnalysisResult.reactions` と `memberEndForces`、応答スペクトルは既存ViewModelを利用する。表示選択はFrontend stateだけで保持する。

## 6. 描画ロジック案

モデル座標ベクトルを `v_m=[x,y,z]^T` とする。通常表示は `v_v=I v_m`、SPACER表示は次の表示専用行列を適用する。

```text
P = [1 0 0
     0 0 1
     0 1 0]

v_viewer = P v_model
```

部材ローカル値は既存要素端力を利用し、位置のみ `P` で変換する。値と符号は変換しない。

## 7. 座標変換方針

project.json、Backend payload、解析結果は変更しない。節点位置、反力ベクトル、ラベル位置に限りViewerで変換する。部材ローカルFXラベルは部材i端・j端の表示位置へ配置する。

## 8. 回転成分・モーメント成分の表示方針

- 反力RFX/RFY/RFZは全体座標系正方向を正とする。
- 部材端FXは既存ViewModelの断面力規約を使用し、i端は要素端力の符号を断面力へ変換済み、j端は既存値を使用する。
- RMX/RMY/RMZとMX/MY/MZの円弧矢印はフェーズ2。
- 未決事項: 右手系円弧の視点依存表現、部材ローカルy/z軸の常時表示、重複ラベル回避、単位表示切替。

## 9. テスト方針

値整形、符号、成分選択、SPACER表示時のラベル位置、ゼロ・非有限値ガードを純粋関数とThree.jsオブジェクト生成で検証する。

## 10. 実装フェーズ分割案

1. RFX/RFY/RFZと部材端FXの数値ラベル。
2. 全成分ラベル、モーメント矢印、ローカル軸凡例。
3. 分布図、カラーマップ、ラベル衝突回避、SPACER比較検証。

## 11. やらないこと

解析ロジック、API、スキーマ、単位系、帳票、結果保存形式は変更しない。

## 12. SPACER出力との比較観点

荷重ケース、端部、座標系、符号、単位、丸め桁、ゼロ判定を同一条件で比較する。SPACER表示切替は位置変換だけであり、数値符号を変更しないことを確認する。
