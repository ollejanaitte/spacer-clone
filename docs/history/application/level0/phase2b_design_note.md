# Phase 2B 設計メモ: 比較モード

## 設計日時

2026-06-21

## 目的

入門編でのA/B比較機能の設計

## 1. 比較モード案

### 概要

同じ橋のパラメータを変更して、解析結果を比較する機能

### 比較パターン

1. **橋長比較**: 短い橋 vs 長い橋
2. **橋脚高さ比較**: 低い橋脚 vs 高い橋脚
3. **橋脚本数比較**: 1本 vs 2本 vs 3本
4. **荷重比較**: 軽い荷重 vs 重い荷重

### UI要素

- パラメータ入力エリア（A案/B案それぞれ）
- 解析実行ボタン
- 比較結果テーブル
- グラフ比較（オプション）

## 3. 最大変位比較

### 表示内容

- 各節点の最大変位
- A案 vs B案の差分
- 差分のパーセンテージ

## 4. 固有周期比較

### 表示内容

- 1次〜3次固有周期
- A案 vs B案の差分
- モード形状の比較

### 比較指標

- 周期比（B案/A案）
- 変化率（%）

## 5. 反力比較

### 表示内容

- 各支点の反力
- A案 vs B案の差分
- 最大反力の比較

### 比較指標

- 反力比
- 変化率（%）

## 6. 想定リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 解析時間の増加 | 中 | 非同期処理 |
| UIの複雑化 | 中 | タブ切り替え |
| 結果の見分け難 | 低 | 色分け |
| メモリ使用量 | 低 | 結果のキャッシュ |

## 7. 実装方針

### 段階的実装

1. **Phase 2B-1**: A/Bパラメータ入力UI
2. **Phase 2B-2**: 2つの解析を並列実行
3. **Phase 2B-3**: 比較結果テーブル
4. **Phase 2B-4**: グラフ比較（オプション）

### 技術的な考慮

- 既存の解析APIを再利用
- 結果は一時的にキャッシュ
- 比較結果は新しいコンポーネントで表示
