## 0. 全体方針
- 本画面は「読み込み → 確認 → 実行 → 結果」の一方向ガイドに専念。
- モデル定義/支点条件/質量設定など前段で確定済みの項目は本画面では編集不可とし、
  操作ボタンは非表示にする (表示のみのステータスカードに変更)。
- カラー・スペーシング・タイポグラフィを整理し、ダッシュボード調の UI に刷新。

## 3. 解析条件設定画面の刷新 (問題③)
### 3.1 レイアウト
```
[① 基本条件 (必須)]
  - 質量ケース [select]   - 地震波 [select]
  - 入力方向 [X/Y/Z]

[② 時間設定]
  - dt [秒]      ☐ 地震波の dt に合わせる
  - 解析時間 [秒] ボタン「地震波長に合わせる」

[③ 減衰設定] (アコーディオン)
  - Rayleigh α / β
  - プリセット: 標準 (h=0.05)

[④ Newmark β法] (折り畳み・固定値)
  β = 0.25 / γ = 0.5

[プレビュー: dt × N = 解析時間 49.99s, 点数 5000]
[この条件を保存]   [解析実行へ進む]
```
### 3.2 バリデーション
- dt と地震波 dt の不一致 → 黄色警告 + 自動補正ボタン
- 解析時間 > 地震波長 → 赤色警告 + 自動補正ボタン

## 5. 時刻歴グラフ不具合 (問題⑤)
### 5.1 症状
ステップ 7 / 結果ページ 3 で線がフラット表示 (値オーダー 1e-6 で Y 軸が潰れている)。

### 5.2 改修
- Recharts の YAxis を `domain={['auto','auto']}` にし、
  `tickFormatter` で `Number(v).toExponential(2)` を採用。
- `|y| < 1e-3` の場合は指数表記 + 単位スケール (μ, m, k) を自動切替。
- X軸: 0〜解析時間、グリッド 5 本、ホバーで時刻と値を表示。
- 表示対象を「応答キー × 物理量 (変位/速度/加速度)」で複数選択可。
- 凡例クリックで系列の表示 ON/OFF。
- 「画像保存(PNG)」「CSV出力」ボタンを追加。

### 5.3 受け入れ条件
- G3_ux 変位 (max 1.08e-5) が判別可能に描画される。
- G2_ux 加速度 (max 0.95) も同一画面で問題なく描画される。

## 6. 3D アニメーション復旧 (問題⑥)
### 6.1 UI 要素
- キャンバス: react-three-fiber、XYZ軸 (赤/緑/青)、グリッド、視点リセット
- 表示要素切替: [節点] [部材] [節点ラベル] [部材ラベル] [支点記号] [質量]
- サイズ調整: 節点サイズ 1〜10px / 部材線幅 1〜6px (スライダ)
- 再生コントロール:
  - ▶ 再生 / ⏸ 一時停止 / ⏮ リセット
  - シークバー (現在時刻 0〜解析時間)
  - 再生速度: 0.25× / 0.5× / 1× / 2× / 4×
  - 変形倍率: 1 / 10 / 100 / 1000 / 任意入力
- 変位モード: 合成 √(x²+y²+z²) / X軸 / Y軸 / Z軸
- ステータス表示: 現在時刻 / 最大変位節点 / 最大値発生時刻

### 6.2 データフロー
```
analysisResult.disp[t][nodeId] = {ux, uy, uz}
  ↓ animationStore { playing, currentT, speed, scale, mode }
  ↓ THREE.Mesh.position をフレーム毎に更新
```

### 6.3 受け入れ条件
- 結果取得後にアニメーションタブを開くとモデルが描画される。
- 再生で節点が変位、シークで任意時刻にジャンプ。
- 変形倍率 1000 で 1e-5 オーダーの変位が視認可能。
